I implemented the search-and-filter feature by adding a toolbar above the board with text and dropdown filters for status, priority, assignee, and overdue state. The board now updates from the API based on the active filters, and when no results match, it shows a clear empty-state message instead of leaving the view blank. For the due-date feature, I added a due-date field in the task form, displayed it on cards, and showed visual reminders for overdue or soon-due tasks.

The main alternatives AI suggested were more complex versions of the same ideas, such as adding a separate advanced filter panel, supporting a larger number of filter combinations with custom operators, and showing a full modal or popover for reminders. I rejected those because they would make the interface heavier and less focused on the current Kanban experience. I also avoided adding backend logic that was beyond the scope of the existing in-memory task tracker, since the simpler UI and API-based approach already met the requested behavior clearly.
